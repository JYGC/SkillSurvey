# pocketbaseserver — Codebase Index

## Purpose

PocketBase-based backend replacing the old `backend/` stack. Provides:
- Auto-generated REST API for all collections
- User authentication (email/password)
- Row-level security via PocketBase rules
- Serves the compiled frontend from `./pb_public`

## Tech stack

- Go 1.24.1
- PocketBase v0.29.1 (BaaS framework + embedded SQLite)
- `modernc.org/sqlite` — pure-Go SQLite driver (no CGO, OpenBSD-compatible)
- Target OS: OpenBSD

## Collections

All collections are served under `/api/collections/<name>/records`.

### sites
| Field | Type | Notes |
|---|---|---|
| `name` | Text, required | e.g. "au.jora.com" |
| `url` | Text, required | Base URL |

Access: publicly readable.

### skillTypes
| Field | Type | Notes |
|---|---|---|
| `name` | Text, required | Category name |
| `description` | Text, required | Category description |

Access: public.

### skillNames
| Field | Type | Notes |
|---|---|---|
| `name` | Text, required | Skill name |
| `isEnabled` | Bool | Include in reports |
| `skillType` | Relation → skillTypes, required | |

Access: public.

### skillNameAliases
| Field | Type | Notes |
|---|---|---|
| `skillName` | Relation → skillNames, required | |
| `alias` | Text, required | Alternative search term |

Access: public.

### jobPosts
| Field | Type | Notes |
|---|---|---|
| `jobSiteNumber` | Text, required | Job ID on source site |
| `site` | Relation → sites, required | |
| `content` | JSON, required | Full job post (title + body) |
| `postedDate` | Date | When posted |
| `location` | JSON, required | `{city, country, suburb}` |

Access: publicly readable.

### monthlyCountReports
| Field | Type | Notes |
|---|---|---|
| `identifier` | Text, required | `<skillNameID>_<YearMonth>` |
| `YearMonth` | Text, required | YYYY-MM |
| `yearMonthDate` | Date, required | First day of period |
| `count` | Number (int), required | Occurrences in that month |
| `skillName` | Relation → skillNames, optional | |

Access: public.

### userSettings
| Field | Type | Notes |
|---|---|---|
| `user` | Relation → users, required, unique | Cascade delete |
| `portalTheme` | Select, required | "white" \| "g10" \| "g90" \| "g100" |

Access: `@request.auth.id != "" && user = @request.auth.id` (own record only).

### users (system)
Standard PocketBase auth collection (email + password).

### Collection dependency order
```

skillTypes

    ├── skillNameAliases
    └── monthlyCountReports
```

## Custom routes

Only one custom route is added:

```go
se.Router.GET("/{path...}", apis.Static(os.DirFS("./pb_public"), false))
```

Serves the compiled frontend SPA. All other endpoints are auto-generated by PocketBase.

## Migrations

Single migration file `1743465600_init_collections.go`:
- `up`: creates all 7 collections in dependency order
- `down`: deletes them in reverse order

Auto-migration is enabled only during `go run` (development). The compiled binary requires explicit `migrate` command or runs migrations on `serve`.

## Build & run

```bash
# Development (auto-migrate, hot-reload errors to console)
make run_dev       # builds then: ./build/pocketbaseserver serve --dev

# Production build
make build         # → ./build/pocketbaseserver

# Run production binary
./build/pocketbaseserver serve
```

Default port: **8090**  
Admin dashboard: `http://localhost:8090/_/`  
REST API base: `http://localhost:8090/api/`

On first launch, visit the dashboard to create a superuser account. The `pb_data/` directory (SQLite + uploads) is created automatically and is git-ignored.

## Frontend static serving

Build the frontend (`npm run build`) and copy the `dist/` output to `pocketbaseserver/pb_public/`. The PocketBase server will serve it at `/`.

## Model accessors

Each file in `internal/models/` wraps `core.BaseRecordProxy` and exposes typed getters/setters:

```go
// Example: skillname.go
func (m *SkillName) Name() string        { return m.GetString("name") }
func (m *SkillName) SetName(v string)    { m.Set("name", v) }
func (m *SkillName) IsEnabled() bool     { return m.GetBool("isEnabled") }
```
